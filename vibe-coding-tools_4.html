// 01 · 先搞懂這件事

Vibe Coding 是什麼？

Vibe Coding 由 AI 研究員 Andrej Karpathy 在 2025 年 2 月提出，原意是：「完全信任 AI，用自然語言描述你要的東西，讓 AI 寫 code，你自己不需要讀或維護那些程式碼。」他把它比喻成「完全沉浸在氛圍裡開發，忘記 code 本身的存在」。

這個概念之所以引發共鳴，是因為它說出了一個真實的現象：越來越多人——包含完全沒有工程背景的創辦人、PM、設計師——開始用 AI 工具把想法直接變成可運行的軟體，速度快到讓傳統開發流程相形見絀。YC 2025 冬季班有 25% 的新創，其 codebase 超過 95% 由 AI 生成。

⚠️ 但「Vibe Coding 工具」這個詞被嚴重濫用了。它被同時用來描述：在對話框請 ChatGPT 寫一段 code、用 Lovable 一鍵出 MVP、用 Cursor 加速日常開發、用 Manus 讓 AI 自己上網完成任務、用 OpenClaw 在手機上傳訊息指揮電腦……這幾件事的本質差異極大，受眾不同、技術門檻不同、能做到的事也完全不同。不搞清楚邊界，很容易拿錯工具，或對工具有不切實際的期待。
// 02 · 我們怎麼分類

分類框架：兩個軸

這份文件用 兩個軸 來整理所有工具，而不是把它們丟進同一個清單裡。

主軸：AI 的自主程度有多高？從「你問它答，每步都需要你在場」，一路到「你說目標，它自己在後台把事情搞定，完成才回報你」。這個維度決定了你需要投入多少時間和注意力，也決定了工具的複雜度和風險程度。

副軸：你需要多少技術背景？主線工具（Foundation Models → App Builders → Autonomous Agents → 編排框架）基本上不需要你懂程式。Dev Tools 這條線則完全不同——它是給本來就在寫 code 的工程師用的加速器，和主線受眾根本不同，所以獨立出來。

💡 多情境通用的工具確實存在。Claude 同時出現在 Foundation Models（對話框請它寫 code）和 Dev Tools（Claude Code 整合進你的 terminal 和 IDE）。ChatGPT 也是。這不是重複，而是同一個底層能力在不同使用方式下扮演不同角色——理解這件事，才能真正用好這些工具。

Foundation Models
純生成工具——整個生態的底層引擎

這一層是整個 Vibe Coding 生態的根基。ChatGPT、Claude、Gemini 在對話框裡可以幫你寫 code、解釋邏輯、Debug、拆解需求——但它們本身沒有執行環境。你拿到的是文字和 code 片段，要在哪裡跑、怎麼部署，仍然是你的責任。

這不代表它們「比較弱」。幾乎所有上面介紹的工具——Lovable、Cursor、Replit、Manus——都建在這些模型之上。理解它們的能力和限制，才能真正理解整個生態的邊界在哪裡。

💡 這一層也是最容易上手的 Vibe Coding 入口。很多人的第一次體驗，就是在 ChatGPT 或 Claude 的對話框裡貼一段需求，然後把生成的 code 貼到別的地方跑。這完全合理——只是要清楚它的限制在哪裡，以及什麼時候該升級到下一層工具。

Autonomous Agents
它自己操作電腦完成任務

這一層是 Vibe Coding 概念的自然延伸，也是目前發展最快、爭議最多的領域。在進入工具介紹之前，有幾個概念需要先說清楚。

什麼是真正的 Agent？和「一般 AI 對話」差在哪？

一般的 AI 對話模式是：你問 → AI 回答 → 你看完決定下一步 → 你再問。每一步都需要你在場，你是執行者，AI 是建議者。

Agent 的模式是：你說目標 → AI 自己拆解需要哪些步驟 → 自己執行每一步 → 遇到問題自己想辦法 → 全部完成後才回報你。AI 變成執行者，你變成下指令的人。

關鍵在於：真正的 Agent 有自己可以操作的真實電腦環境——真實的瀏覽器、terminal、檔案系統。它可以自己上網查資料、寫程式然後執行、寄信、填表單，你不在場它也在跑。這和 App Builder 裡「自動規劃生成 code 的 Agent Mode」是根本不同的東西。
雲端 Agent vs 本機 Agent：這不只是技術細節，而是資料主權的選擇。

雲端 Agent（如 Manus 網頁版）在廠商的伺服器上執行任務。不需要任何設定，開即用，但你的資料、任務內容、執行過程都在廠商的基礎設施上。

本機 Agent（如 Manus 本機版、OpenClaw）在你自己的電腦上執行。資料完全在你手上，不經過任何第三方伺服器，但你需要自己安裝設定，技術門檻明顯更高，安全風險也需要你自己負責管理。

選哪個，取決於你對資料隱私和安全的要求，以及你願意投入多少設定成本。

Agent 編排框架
管理多個 Agent 的組織架構層

一個 Agent 幫你執行任務，已經很強大了。但現實中複雜的業務場景，往往需要多個專職 Agent 分工協作：一個負責研究、一個負責寫程式、一個負責審查、一個負責對外溝通……這就需要一個「管理層」來協調它們——定義誰負責什麼、它們怎麼溝通、預算怎麼分配、出了問題誰負責。

這一層工具的核心不是執行任務，而是定義組織架構。如果說 OpenClaw 是一個 AI 員工，Paperclip 就是讓你建立和管理整個 AI 公司的框架。

💡 這是目前生態裡最早期、最前沿的一層。使用者需要對 Agent 架構有一定理解，不適合作為 Vibe Coding 的入門點。但如果你已經在思考「如何讓多個 AI 系統協作完成複雜業務流程」，這是你需要關注的方向。

這條線和主線的本質差別：你還是主導者，AI 是你的共同作者。你需要有程式背景才能有效使用這些工具，因為你要能判斷 AI 生成的 code 是否正確、架構是否合理、什麼時候該推翻重來。

那為什麼它們會出現在 Vibe Coding 的語境裡？因為它們代表了一個現象：有程式背景的人，現在也越來越多在「用說的」來寫 code——用自然語言描述要做什麼，讓 AI 執行跨多個檔案的複雜改動，自己 review 結果。這和傳統的「AI 補全」體驗已經完全不一樣了。

💡 多情境提醒：Claude 和 ChatGPT 在這裡再次出現，但角色完全不同。對話框裡的 Claude 是 Foundation Model；Claude Code 是有工具鏈整合、可以直接操作你 repo 的 agent 工具。同一個底層模型，不同的使用深度和方式——這也是理解整個 Vibe Coding 生態最重要的一個概念。

Who Should Use What
🚀
非技術創辦人，想快速驗證 MVP
從 ChatGPT / Claude 開始拆解需求，然後用 Lovable 直接建出來。最快路徑。
🎨
要做高品質 React / Next.js 前端
v0。產出最乾淨，直推 Vercel，UI 品味最高。
🏗️
需要完整後端 + 零設定
Base44。Auth、DB、API 一條龍，zero-shot 最完整。
📚
初學者 / 學生入門
Replit。50+ 語言、社群最豐富，瀏覽器寫 code 並直接部署。
🕹️
想讓 AI 自己去完成任務，不想全程盯著
Manus（雲端）。研究、分析、多步驟任務交給它，你去做別的事。
🔐
想要 Agent 能力，但資料不能離開本機
Manus（本機） 穩妥；OpenClaw 開源但安全風險高，需要技術能力評估後使用。
💼
專業開發者想大幅提速
Cursor 是日常主力；大型 codebase 重構首選 Claude Code。
🏢
想讓多個 AI Agent 分工協作
Paperclip。最前沿但最早期，適合已理解 Agent 架構的人。
🔄
最多人在用的黃金工作流
Lovable 快速原型驗證 → Cursor 重構上生產。兩者互補不競爭。
⚠️
真實警告，讀完再決定
研究顯示 AI 工具讓有經驗的開發者慢了 19%。感覺快 ≠ 真的快。你還是要能看懂 AI 生出來的東西。
